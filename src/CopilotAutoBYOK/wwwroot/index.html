连接配置
API endpoint URL，激活 BYOK 模式
openai（默认）/ azure / anthropic
API 密钥，本地提供商（如 Ollama）可省略
优先级高于 API key
completions（默认）或 responses，GPT-5 系列用 responses
Azure API 版本，默认 versionless v1
模型配置
代理层使用的模型名称，同时作为 MODEL_ID 的默认值
已知模型 ID，用于 token 限制和工具支持。留空自动使用 COPILOT_MODEL 的值
发送给提供商 API 的模型名称（如 Azure deployment）。默认与 MODEL_ID 相同
最大 prompt tokens。当模型不在 CLI 内置目录时必须设置
最大输出 tokens。当模型不在 CLI 内置目录时必须设置
快速示例
Ollama（本地，无需 API key）
COPILOT_PROVIDER_BASE_URL=http://localhost:11434/v1 \
  COPILOT_MODEL=deepseek-coder-v2:16b \
  copilot
自定义 OpenAI 兼容端点
COPILOT_PROVIDER_BASE_URL=https://my-api.example.com/v1 \
  COPILOT_PROVIDER_API_KEY=sk-... \
  COPILOT_MODEL=gpt-4 \
  copilot
Azure OpenAI（自定义 wire model）
COPILOT_PROVIDER_TYPE=azure \
  COPILOT_PROVIDER_BASE_URL=https://my-resource.openai.azure.com \
  COPILOT_PROVIDER_API_KEY=your-key-here \
  COPILOT_PROVIDER_MODEL_ID=gpt-4 \
  COPILOT_PROVIDER_WIRE_MODEL=my-gpt4-deployment \
  copilot
Anthropic
COPILOT_PROVIDER_TYPE=anthropic \
  COPILOT_PROVIDER_BASE_URL=https://api.anthropic.com \
  COPILOT_PROVIDER_API_KEY=sk-ant-... \
  COPILOT_MODEL=claude-sonnet-4-20250514 \
  copilot

调用示例 

curl http://localhost:5267/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "auto-copilot",
    "messages": [{"role": "user", "content": "你好！"}]
  }'
curl http://localhost:5267/v1/messages \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "auto-copilot",
    "messages": [{"role": "user", "content": "你好！"}],
    "max_tokens": 1024
  }'
